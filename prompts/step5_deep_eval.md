# STEP 5: LLM深層評価

## 何が分かるか

感情語の「額面通りの意味」と「物語上の機能」のズレ。「笑ってるけど悲しい」「怒ってるけど心配」のような深層構造を数値化する。

## プロンプト（1話ずつ実行推奨）

```
以下は小説の第N話です。この話の感情表現を「表層」と「深層」の二重構造で分析してください。

■ 表層分析：
テキストに直接現れている感情語を抽出し、
7カテゴリ（喜び/悲しみ/不安/怒り/驚き/覚悟/静寂）に分類。
各カテゴリの強度を1-10で評価してください。

■ 深層分析：
それぞれの感情語が「物語の中で果たしている機能」を評価してください。
例：
- 「笑う」が取り繕いとして機能 → 表層は喜び、深層は不安
- 「怒り」が心配の裏返し → 表層は怒り、深層は愛情
- 「静かだ」が押し殺しの結果 → 表層は静寂、深層は悲しみ
各カテゴリの深層機能の強度を1-10で評価してください。

■ Gap（乖離度）：
表層と深層の差を0-10で算出してください。
Gapが大きいほど「表面と裏の落差」が大きい話。

■ key_gaps：
特に顕著な表層⇔深層のズレを3つ程度、具体的に記述してください。

■ 出力形式：JSON
{
  "episode": N,
  "surface": {"joy":X, "sadness":X, "anxiety":X, "anger":X, "surprise":X, "resolve":X, "silence":X},
  "deep": {"joy":X, "sadness":X, "anxiety":X, "anger":X, "surprise":X, "resolve":X, "silence":X},
  "gap": X,
  "key_gaps": ["具体的な乖離の説明1", "説明2", "説明3"]
}

（ここに1話分のテキストを添付またはペースト）
```

## 使い方のコツ

- **1話ずつ実行する**（全話一括だと精度が落ちる）
- 結果のJSONを全話分蓄積し、最後にまとめて可視化する
- Gap ≥ 7 の話は「読者に伝わらないリスク」がある。意図的ならOK、そうでないなら推敲ポイント
- Gap ≤ 2 の話は「表裏がない」＝ ストレートに感情が出ている回

## まとめて可視化するプロンプト

全話分のJSONが揃ったら：

```
以下は小説全N話の深層評価JSONです。
これを可視化するHTMLダッシュボードを作ってください。

■ 可視化内容：
1. 表層 vs 深層のレーダーチャート（各話または選択話）
2. Gap（乖離度）の推移折れ線
3. 7カテゴリの表層/深層比較ヒートマップ
4. key_gapsのハイライト一覧

（ここにJSON配列を貼り付け）
```
